Privacy Policy — WHOOP + Sleep8 Personal Automation

Last updated: 2026-04-18

This application is a personal automation and experimentation project that reads WHOOP health and recovery data and may use that data to generate recommendations for Eight Sleep / Sleep8 pod settings, including temperature adjustments and potentially elevation-related recommendations.

What data may be accessed

  • WHOOP recovery-related metrics, including heart rate, resting heart rate, HRV, sleep, and related wellness metrics that are explicitly granted through WHOOP permissions.
  • Sleep8-related local automation state, command outputs, and local logs needed to test or apply recommended bed-setting adjustments.

How data is used

  • To analyze wellness and recovery signals from WHOOP.
  • To generate Sleep8 temperature recommendations.
  • To optionally support future Sleep8 automation flows after explicit validation and approval.
  • To store limited local history needed for debugging, dry-run validation, and improving recommendation logic.

Data storage

Project data may be stored locally in workspace files and local databases for testing, debugging, and iteration. This project is intended for personal/internal use, not a public consumer service.

Data sharing

This project is not intended to sell or share personal health data with third parties. Data is used for personal automation and local testing unless explicitly configured otherwise by the project owner.

Automation safety

By default, the project is expected to operate in dry-run or recommendation mode before any live pod-setting changes are enabled. Live control should only be enabled after validation and explicit user approval.

Access revocation

WHOOP access can be revoked by removing the app's access through WHOOP account/developer controls and by deleting locally stored credentials or tokens from the project environment.

Contact

For this personal/internal project, contact is handled directly by the project owner through the associated GitHub repository.
